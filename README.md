# スピーチフロー可視化ツール for Research Conference

カンファレンスのセッション中にリアルタイム音声書き起こしテキストを入力すると、OpenAI APIがトピック単位に構造化し、Miro への転記に適した形式で可視化する Web アプリケーションです。

## 概要

発表を聴きながら Zoom などの書き起こし機能で得たテキストを 2〜5 分間隔で貼り付けると、AI がトピックを抽出して 3 種類のノードとして整理します。生成されたノードは Supabase を介してリアルタイムに同期され、複数ユーザーで同一セッションを共有・閲覧できます。

```
音声書き起こし → テキスト入力 → AI 解析・ノード生成 → Miro へ転記
```

## 機能

### ノード自動生成
テキストを入力するたびに、以下の 3 種類のノードを自動生成します。

| ノード種別 | スタイル | 用途 |
|------------|----------|------|
| タイトル   | 水色背景 + 青枠 + 太字 | トピックの主題 |
| 事実       | 白背景 + 青い点線枠 | 客観的情報・データ |
| 洞察       | #3640A5 背景 + 白文字 | 主観的考察・分析 |

- 1 ノードあたり 80 文字以内（超過時は自動分割）
- 情報量レベル・テキスト密度をユーザーが調整可能

### セッション管理
- カンファレンス単位でセッションを作成・管理
- Supabase によるリアルタイム自動保存・同期
- 他ユーザーのセッションを読み取り専用で閲覧可能
- オフライン時はローカルストレージにバックアップし、復旧後に自動同期

### 感想・質問生成
セッション全体の内容から OpenAI API が感想コメントと質疑応答用の質問を自動生成します。カンファレンス後の討議・フォローアップに活用できます。

## 技術スタック

- **フレームワーク**: React 18 + TypeScript + Vite
- **UI**: Tailwind CSS v4 + shadcn/ui (Radix UI)
- **AI**: OpenAI API (デフォルト: gpt-4o)
- **バックエンド / DB**: Supabase (Edge Functions + リアルタイム同期)
- **パッケージマネージャー**: pnpm

## セットアップ

### 前提条件
- Node.js 18 以上
- pnpm
- OpenAI API キー
- Supabase プロジェクト（本リポジトリのデフォルト設定を使用するか、独自に作成）

### インストール

```bash
pnpm install
```

### 開発サーバー起動

```bash
pnpm dev
```

ブラウザで `http://localhost:5173` を開きます。

### ビルド

```bash
pnpm build
```

## 設定

アプリ右上の設定ボタンから以下を入力します。

| 設定項目 | 説明 |
|----------|------|
| OpenAI API キー | テキスト解析・感想生成に使用 |
| AI モデル | 使用するモデル（デフォルト: `gpt-4o`） |
| Supabase URL | セッション保存先エンドポイント |
| Supabase Anon Key | Supabase 認証用の公開キー |

設定値は `localStorage` に保存され、次回以降も維持されます。

## 使い方

1. セッションを作成（左サイドバー「＋」ボタン）
2. Zoom 等の書き起こしテキストを 2〜3 分ごとにテキストエリアへ貼り付け
3. 「追加」ボタンで送信 → キャンバスにノードが生成される
4. 随時キャンバスの内容を Miro へ転記してレイアウト調整
5. 「感想・質問生成」ボタンでセッション全体のフィードバックを取得

## オフラインモード

サーバーへの接続に失敗した場合は自動的にオフラインモードへ切り替わります。

- データは `localStorage` に保存されます
- ネットワーク復旧後、自動的にサーバーと同期します
- OpenAI API が利用できない場合はフォールバック処理で基本的なノード生成を継続します
