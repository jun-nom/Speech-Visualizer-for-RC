# スピーチフロー可視化ツール - チュートリアル

## はじめに

このチュートリアルでは、カンファレンス配信やYoutube Liveの音声書き起こしテキストを構造化し、Miroでの可視化に活用するための実践的な手順を説明します。リアルタイム配信を想定した実際のワークフローを詳しく解説します。

## ステップ1: 初期設定

### 1-1. アプリケーションへのアクセス
1. ブラウザでアプリケーションにアクセスします
2. 画面が表示されたら、右上の⚙️（設定）ボタンをクリックします

### 1-2. OpenAI API設定（必須）
1. 設定ダイアログが開いたら「OpenAI API Key」欄にAPIキーを入力します
   - OpenAI APIキーをお持ちでない場合は、[OpenAI公式サイト](https://openai.com/)で取得してください
2. 「AI Model」は初期設定の「gpt-4o」のままで問題ありません
3. 「保存」ボタンをクリックして設定を保存します

### 1-3. 設定完了確認
- 設定が完了すると、自動的に新しいセッションが作成されます
- 左サイドバーに現在時刻をタイトルとしたセッションが表示されます

## ステップ2: カンファレンス配信の構造化開始

### 2-1. ノートPC環境での配信セットアップ
1. **準備するツール**:
   - カンファレンス配信（Youtube Live、Zoom等）
   - 音声書き起こしツール（Zoomの書き起こし機能等）
   - 本アプリケーション
   - Miroボード

2. **ノートPCでの効率的な画面配置**:
   - **方法1（タブ切り替え）**: 各ツールを別タブで開き、Ctrl+Tab等で高速切り替え
   - **方法2（画面分割）**: 配信視聴を左半分、本アプリを右半分に分割表示
   - **方法3（最小化活用）**: 作業中のツール以外は最小化し、タスクバーから切り替え

3. **推奨ブラウザタブ順序**:
   1. カンファレンス配信
   2. 音声書き起こしツール  
   3. 本アプリケーション
   4. Miroボード

### 2-2. 最初のテキスト入力（配信開始2〜3分後）
1. 音声書き起こしツールから最初の2〜3分間のテキストをコピーします
2. 本アプリの「スピーチ内容を入力してください...」欄にペーストします

**例: カンファレンス冒頭の書き起こしテキスト**
```
皆様、本日はデジタル・トランスフォーメーション・カンファレンス2024にご参加いただき、ありがとうございます。私は今日、AIを活用したビジネス変革について3つの観点からお話しします。まず現在の市場動向、次に実践的な導入事例、最後に今後の展望についてご紹介いたします。
```

### 2-3. 最初のフロー生成
1. 「フローに追加」ボタンをクリックします
2. 処理完了後、メインエリアに以下のようなノードが表示されます：
   - **タイトルノード**: 「DXカンファレンス2024開会」
   - **事実ノード**: 「AIビジネス変革を3つの観点で説明：市場動向、導入事例、今後展望」
   - **洞察ノード**: 「体系的なアプローチでAI活用の全体像を提示」

### 2-4. Miroへの最初の転記（ノートPC操作）
1. **タブ切り替え**: Ctrl+Tab（Windows）またはCmd+Option+右矢印（Mac）でMiroタブに移動
2. 生成されたノード内容をMiroボードにコピー&ペースト
3. Miro上でカンファレンスのタイトル部分として配置
4. 今後追加されるコンテンツのスペースを確保
5. **復帰**: 再度Ctrl+Tab等で本アプリタブに戻る

## ステップ3: 配信進行に合わせた継続的な構造化

### 3-1. 第2セクションのテキスト入力（配信開始5〜8分後）
1. 音声書き起こしツールから次の2〜3分間のテキストをコピー
2. トピックの区切りを意識してテキストを選択（発表者の「次に...」「続いて...」などの転換点）

**例: 市場動向セクションの書き起こしテキスト**
```
まず最初に市場動向についてお話しします。2024年のAI市場は前年比で45%の成長を記録しており、特にGenerative AI分野が急速に拡大しています。McKinseyの調査によると、企業の73%がすでに何らかの形でAIを業務に導入しており、ROIは平均で20%向上していることが判明しました。
```

### 3-2. フローの継続的拡張
1. 「フローに追加」ボタンをクリック
2. 新しいトピックのノードセットが追加されます：
   - **タイトルノード**: 「AI市場動向2024」
   - **事実ノード**: 「AI市場45%成長、Generative AI急拡大、企業73%導入済、ROI20%向上」
   - **洞察ノード**: 「AI導入は実験段階から実用段階へ移行、明確なビジネス価値を創出」

### 3-3. ノートPCでのリアルタイムMiro更新
1. **効率的なタブ操作**:
   - 本アプリでノード生成完了
   - Ctrl+Tab等でMiroタブに切り替え
   - 新しく生成されたノード内容をMiroに追加
   - カンファレンスの進行に合わせてレイアウト調整
   - 再び本アプリタブに戻る

2. **省エネ操作テクニック**:
   - Miroでの詳細レイアウト調整は配信の休憩時間に集中
   - 配信中は素早くコピー&ペーストのみに集中
   - トピック間の関連性付けは後からまとめて実施

### 3-4. 継続的な配信フォロー（ノートPC最適化）
- **基本サイクル（2〜3分間隔）**:
  1. 配信視聴タブ → 書き起こしツールタブ（テキストコピー）
  2. 本アプリタブ（テキスト入力・処理実行）
  3. Miroタブ（結果の転記）
  4. 配信視聴タブに戻る
- **休憩時間活用**: セクション終了時にMiroレイアウトを整理
- **長時間配信対応**: 5分間隔での入力も可、画面疲労を考慮

## ステップ4: 感想・質問の生成

### 4-1. フィードバック生成
1. 十分なテキストを入力した後、「感想・質問を生成」ボタンをクリックします
2. 処理中はボタンが「生成中...」と表示されます
3. 完了すると右サイドバーに以下が表示されます：
   - **感想**: スピーチ内容に対する建設的なコメント
   - **質問**: より深い議論を促進する質問

### 4-2. フィードバックの活用例
**表示される感想の例:**
- 「具体的な数値データを示すことで説得力のある報告になっています」
- 「ターゲット拡大戦略が明確で実現可能性が高そうです」

**表示される質問の例:**
- 「30代男性向けマーケティングの予算配分はどのように考えていますか？」
- 「競合他社の動向についてはいかがでしょうか？」

## ステップ5: セッション管理

### 5-1. 新しいセッションの作成
1. 左サイドバーの「新規セッション」ボタンをクリックします
2. 新しいセッションが作成され、クリーンな状態から開始できます
3. 以前のセッションは自動的に保存されます

### 5-2. セッション間の切り替え
1. 左サイドバーのセッション一覧から任意のセッションをクリックします
2. 選択したセッションの内容が表示されます
3. 過去のスピーチフローを確認・編集できます

### 5-3. セッションの削除
1. 不要なセッションの右端にある「🗑️」アイコンをクリックします
2. 確認ダイアログが表示されるので「削除」をクリックします
3. セッションが完全に削除されます

## ステップ6: 実践的な使用シナリオ

### 6-1. テックカンファレンス配信
**シナリオ**: 3時間のAI・機械学習カンファレンス
**使用方法:**
1. 各講演セッション（20〜30分）を2〜3分間隔で構造化
2. 講演者の交代時にMiroでセクション区切りを調整
3. 休憩時間中にこれまでの内容を整理・レイアウト調整

**想定される入力例:**
```
続いて、機械学習モデルの実装について技術的な観点から説明します。
まず、データ前処理において最も重要なのは特徴量エンジニアリングです。
我々の実験では、適切な特徴量選択により精度が15%向上しました...
```

### 6-2. ビジネスウェビナー
**シナリオ**: 1時間のマーケティング戦略ウェビナー
**使用方法:**
1. ウェビナーの各トピック（10〜15分）を段階的に構造化
2. 質疑応答セッションも含めて記録
3. 最終的にMiroで全体の戦略フローとして整理

### 6-3. 学術シンポジウム
**シナリオ**: 半日の研究発表シンポジウム
**使用方法:**
1. 各研究発表（15〜20分）の要点を構造化
2. パネルディスカッションの論点整理
3. Miroで研究分野ごとのクラスタリング表示

## ステップ7: 高度な活用テクニック

### 7-1. 音声書き起こしテキストの効果的な分割
**推奨方法:**
- 発表者の明確な話題転換点で区切る（「次に...」「続いて...」「また...」）
- 1回の入力は200〜500文字程度（2〜3分間の書き起こし相当）
- 質疑応答セッションは質問と回答を別々に入力
- 複数の発表者がいる場合は発表者ごとに区切る

### 7-2. ノートPC環境でのMiro連携最適化
**効率的な転記方法:**
- **クイック転記**: ノード生成後、即座にCtrl+Tabでタブ切り替えしてMiroにコピー
- **バッチ処理**: 複数ノード生成後にまとめてMiroに転記（タブ切り替え回数を削減）
- **キーボードショートカット活用**: 
  - Ctrl+C（コピー）、Ctrl+Tab（タブ切り替え）、Ctrl+V（ペースト）の高速操作
  - Ctrl+1、Ctrl+2等でタブ位置指定切り替え
- **段階的レイアウト**: 配信中は簡単配置、休憩時間に詳細調整
- **ブックマーク活用**: よく使うMiroボードはブックマークして素早くアクセス

### 7-3. リアルタイム配信での注意点
**効率的な運用方法:**
- 音声書き起こしの精度を事前に確認（固有名詞や専門用語の認識）
- 配信の音質が悪い場合は手動でテキスト補正
- 重要な数値や固有名詞は書き起こし後に再確認
- 配信中断や接続問題に備えてローカル保存を活用

### 7-4. 生成された感想・質問の活用
**配信後の活用方法:**
- カンファレンス参加者への追加情報提供の参考
- 次回開催時の改善点として記録
- 関連セッションや続編企画のアイデアソース
- 参加者アンケートの質問項目として活用

## トラブルシューティング

### よくある問題と解決方法

**Q: フローが生成されない**
A: 
- OpenAI APIキーが正しく設定されているか確認
- 音声書き起こしテキストの文字数が極端に少なくないか確認
- インターネット接続を確認（カンファレンス会場の回線品質）
- しばらく待ってから再試行

**Q: 音声書き起こしの精度が低い**
A: 
- 配信の音質設定を確認
- 専門用語や固有名詞は手動で修正してから入力
- 発表者の話すスピードが速い場合は少し長めの間隔で区切る

**Q: ノートPCでのタブ切り替えが煩雑**
A: 
- キーボードショートカットを覚える（Ctrl+Tab、Ctrl+Shift+Tab等）
- ブラウザのタブを固定（Pin）して位置を固定
- 使用頻度の高いツールを最初のタブ位置に配置
- マウスよりキーボード操作を優先

**Q: Miroに転記する際のレイアウトがうまくいかない**
A: 
- 配信中は素早く縦一列に配置（詳細は後回し）
- 休憩時間や配信終了後に関連性でグルーピング
- 小さな画面では拡大・縮小を頻繁に使用
- トピック間の関係性を矢印や線で明示

**Q: 長時間配信で処理が重くなる・画面疲労**
A: 
- 1時間程度で新しいセッションに切り替え
- 定期的にブラウザをリフレッシュ
- 不要なタブを閉じてメモリを確保
- 画面の明度調整や休憩を適度に取る

1. **操作効率の向上**: 
   - キーボードショートカットを完全に習得
   - 個人に最適なタブ配置パターンを確立
   - よく使うツールのブックマーク整理

2. **配信別最適化**: 
   - 継続参加するカンファレンスシリーズで一貫した構造化手法を確立
   - Miroテンプレート化（配信形式に合わせたレイアウト）
   - 配信時間に応じた休憩・切り替えパターンの確立

3. **チーム活用**: 
   - 生成されたMiroボードをチームメンバーと共有し、知識共有を促進
   - アーカイブ活用（過去の配信構造化データの検索・参照）
   - フィードバック循環（配信主催者や他の参加者との共有）

4. **ノートPC環境の改善**: 
   - 外部キーボード・マウスの活用検討
   - ディスプレイポートやHDMI接続での画面拡張（可能な場合）
   - エルゴノミクス（姿勢、照明等）の最適化

このツールをノートPC環境で効率的に活用することで、限られた画面スペースでもカンファレンス内容を体系的に構造化し、効果的な知識共有を実現できます。
